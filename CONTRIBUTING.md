# 🤝 Contributing to CalmFlow

CalmFlowへのコントリビューションに興味を持ってくれてありがとうございます！

## 開発環境のセットアップ

特別な環境構築は不要です。

```bash
git clone https://github.com/YOUR_USERNAME/calmflow.git
cd calmflow
# ブラウザでindex.htmlを開くだけ！
```

## コードの構成

`index.html` は1ファイルに全てが含まれています：

```
<style>     → CSS（テーマ、レイアウト、コンポーネント）
<body>      → HTML構造
<script>    → JavaScript（全てのロジック）
```

### JavaScriptの主要な関数

| 関数 | 役割 |
|------|------|
| `rCal()` | カレンダーの描画 |
| `rDV()` | デイリープランナーの描画 |
| `rTodos()` | Todoリストの描画 |
| `rTimer()` | タイマーページの描画 |
| `rChart()` | 棒グラフの描画 |
| `uSt()` | 統計の更新 |
| `applyTheme()` | テーマの適用 |

### データの保存

全てのデータは `localStorage` に保存されます：

| キー | 内容 |
|------|------|
| `cf-ev` | イベント |
| `cf-td` | Todo |
| `cf-slog` | 勉強ログ |
| `cf-subj` | 科目 |
| `cf-cfg` | 設定（言語、テーマ） |
| `cf-sch` | スケジュール |
| `cf-mem` | メモ |
| `cf-goals` | 目標 |

## プルリクエストのガイドライン

1. **1つのPRにつき1つの機能/修正** にしてください
2. 既存の機能が壊れていないことを確認してください
3. コミットメッセージは日本語でも英語でもOKです
4. 新しい機能を追加した場合は、READMEも更新してください

## バグ報告

Issueを作成する際は以下を含めてください：
- 使用ブラウザとバージョン
- 再現手順
- 期待される動作
- 実際の動作
- スクリーンショット（あれば）

## 機能リクエスト

新しい機能のアイデアがあれば、Issueで提案してください！
